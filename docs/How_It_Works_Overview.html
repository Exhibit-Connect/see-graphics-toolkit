How It Works

The SEE graphics process — kickoff to final render

What happens at each stage, and what we built for it. Green = automated now · Grey = still a person's job.
1Kickoff & design
Job is won. The 3D team designs the booth and exports the placement drawings — the PDF (or Illustrator file) that shows every wall and its size.
3D team
›
2Capture the key
intake.py reads those placement drawings and pulls every wall, size and finish into one booth file — and flags any size that disagrees between pages.
intake.py
›
3Tell the client
From that one booth file, auto-build the Illustrator wall templates and the client spec sheet (every size, the bleed, the due date) — identical every time, no hand-building.
template generator + spec sheet
›
4Client submits
The moment artwork comes back, proofer.py checks each file against the booth file: size, color, resolution, fonts, even spelling — and writes a pass/fail report.
proofer
›
5Proof & sign-off
make_proof.py builds a branded proof sheet (the art + the checks) for the client to sign. Approval is dated, locked and logged — and we own that record, not the vendor.
make_proof
›
6Print & render
The approved, verified files go to the print vendor, then back to the designer for the final booth render.
vendor + designer
🔑 The trick: the booth file is defined once — straight from the 3D drawings (stage 2). Stages 3–5 all read from it, so the templates, the client spec sheet, and the artwork checks can never disagree with each other.
