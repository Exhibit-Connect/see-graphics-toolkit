# Examples

A complete worked example — the booth **Mama's Creations – IDDBA 2026** — shown in the order the
real workflow produces it. Open them **1 → 7** to follow the whole pipeline.

| # | File | What it is |
|---|------|-----------|
| 1 | `1_booth_spec_example.json` | The **booth file** — every wall + size. The single source everything else is built from. |
| 2 | `2_intake_checklist.md` | What `intake.py` produces from the 3D handoff: the draft + the items to confirm. |
| 3 | `3_wall_template_preview.png` | What the wall templates look like — generated by `tools/preview_templates.py` from the booth file (no Illustrator). |
| 4 | `4_client_spec_sheet.pdf` | The sheet sent to the client (sizes, rules, due date). |
| 5 | `5_artwork_check_report.pdf` | The proofer's PASS / FAIL report on a submitted file. |
| 6 | `6_client_proof_sheet.pdf` | The client proof for **one graphic** + the dated, locked sign-off. |
| 7 | `7_job_proof_document.pdf` | The **whole-job** proof: a cover/summary page (every graphic, size, material, qty) + one page per graphic + Page X of Y — what the client gets for the full order. |

For reference only — these don't drive anything. For a real job, make a new booth file (see `../docs/Instructions.md`).
